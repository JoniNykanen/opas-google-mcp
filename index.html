Yhdistä Google Analytics ja Search Console AI-assistenttiin

Claude Code + MCP — käytännön opas

GSC ~5 min GA4 ~20 min MCP-protokolla

Tässä oppaassa yhdistämme Google Search Consolen (GSC) ja Google Analytics 4:n (GA4) suoraan Claude Codeen — Anthropicin AI-koodausassistenttiin, joka toimii terminaalissa.

Yhteys tapahtuu MCP-protokollalla (Model Context Protocol). Se on avoin standardi, joka antaa AI:lle mahdollisuuden kutsua ulkoisia työkaluja reaaliajassa.

Voit kysyä AI:lta esim. "Mitkä sivut saivat eniten klikkauksia viimeisen kuukauden aikana?" — ja saat vastauksen suoraan, ilman manuaalista datan kopiointia.

Vaatimukset

Claude Code
npm install -g @anthropic-ai/claude-code
Node.js
Versio 18 tai uudempi
Google-tili
Pääsy GSC:hen ja GA4:ään
gcloud CLI

OSA 1

Google Search Console

GSC-yhteys on helpompi, koska sille on valmis MCP-paketti.

1

Asenna paketti

npm install -g google-searchconsole-mcp
2

Lisää MCP-konfiguraatio

Avaa (tai luo) tiedosto ~/.claude/.mcp.json ja lisää:

{
  "mcpServers": {
    "google-search-console": {
      "command": "google-searchconsole-mcp",
      "args": [],
      "env": {}
    }
  }
}
3

Käynnistä Claude Code

claude

Ensimmäisellä kerralla GSC-paketti pyytää OAuth-kirjautumista. Selain avautuu, kirjaudut Google-tililläsi, ja yhteys on valmis.

Kirjoita Claude Codessa: Listaa kaikki Search Console -sivustoni
Jos näet sivustosi listan, GSC-yhteys toimii.
OSA 2

Google Analytics 4

GA4-yhteys on monimutkaisempi. Useimmat valmiit GA4 MCP -paketit vaativat service account JSON -avaimen. Jos organisaatiosi estää avainten luonnin (kuten monissa yrityksissä), tarvitset vaihtoehtoisen tavan.

Tässä käytämme Application Default Credentials (ADC) -menetelmää ja custom MCP-serveriä.

1

Asenna ja konfiguroi gcloud CLI

Windows (PowerShell) 
winget install Google.CloudSDK
macOS 
brew install google-cloud-sdk
Linux 
curl https://sdk.cloud.google.com | bash

Kirjaudu sisään:

gcloud auth application-default login

Selain avautuu. Kirjaudu Google-tililläsi ja hyväksy oikeudet. Tunnukset tallentuvat paikallisesti.

2

Ota Analytics API käyttöön

Mene Google Cloud Consoleen ja ota Google Analytics Data API käyttöön projektillesi.

3

Selvitä GA4 Property ID

  1. Avaa Google Analytics
  2. Mene Admin → Property Settings
  3. Kopioi Property ID (pelkkä numero, esim. 123456789)
4

Luo custom MCP-serveri

Luo kansio ja asenna riippuvuudet:

mkdir -p ~/.claude/mcp-servers/ga4
cd ~/.claude/mcp-servers/ga4
npm init -y
npm install @google-analytics/data @modelcontextprotocol/sdk zod

Avaa package.json ja lisää kenttä "type": "module" ylätasolle (esim. "name"-kentän jälkeen):

{
  "name": "ga4",
  "type": "module",
  ...
}

Luo tiedosto index.js:

import { BetaAnalyticsDataClient } from "@google-analytics/data";
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const analyticsDataClient = new BetaAnalyticsDataClient();
const PROPERTY_ID = process.env.GA4_PROPERTY_ID || "VAIHDA_OMA_ID";

const server = new McpServer({
  name: "google-analytics",
  version: "1.0.0",
});

// Työkalu: Aja GA4-raportti
server.tool(
  "ga4_run_report",
  "Run a GA4 report with custom dimensions and metrics",
  {
    dateRangeStart: z.string().describe("Start date (YYYY-MM-DD or 'NdaysAgo')"),
    dateRangeEnd: z.string().describe("End date (YYYY-MM-DD or 'today')"),
    metrics: z.array(z.string()).describe("e.g. ['sessions','screenPageViews']"),
    dimensions: z.array(z.string()).optional(),
    limit: z.number().optional(),
  },
  async ({ dateRangeStart, dateRangeEnd, metrics, dimensions, limit }) => {
    const request = {
      property: `properties/${PROPERTY_ID}`,
      dateRanges: [{ startDate: dateRangeStart, endDate: dateRangeEnd }],
      metrics: metrics.map((m) => ({ name: m })),
      dimensions: (dimensions || []).map((d) => ({ name: d })),
      limit: limit || 10,
    };
    const [response] = await analyticsDataClient.runReport(request);
    const headers = [
      ...(response.dimensionHeaders || []).map((h) => h.name),
      ...(response.metricHeaders || []).map((h) => h.name),
    ];
    const rows = (response.rows || []).map((row) => [
      ...(row.dimensionValues || []).map((v) => v.value),
      ...(row.metricValues || []).map((v) => v.value),
    ]);
    return {
      content: [{ type: "text", text: JSON.stringify({ headers, rows }, null, 2) }],
    };
  }
);

// Työkalu: Top-sivut
server.tool(
  "ga4_top_pages",
  "Get top pages by sessions (last 28 days)",
  { limit: z.number().optional() },
  async ({ limit }) => {
    const [response] = await analyticsDataClient.runReport({
      property: `properties/${PROPERTY_ID}`,
      dateRanges: [{ startDate: "28daysAgo", endDate: "today" }],
      metrics: [{ name: "sessions" }, { name: "screenPageViews" }],
      dimensions: [{ name: "pagePath" }],
      orderBys: [{ metric: { metricName: "sessions" }, desc: true }],
      limit: limit || 10,
    });
    const rows = (response.rows || []).map((row) => ({
      page: row.dimensionValues[0].value,
      sessions: row.metricValues[0].value,
      pageViews: row.metricValues[1].value,
    }));
    return { content: [{ type: "text", text: JSON.stringify(rows, null, 2) }] };
  }
);

// Työkalu: Reaaliaikainen data
server.tool(
  "ga4_realtime",
  "Get real-time active users and top pages",
  {},
  async () => {
    const [response] = await analyticsDataClient.runRealtimeReport({
      property: `properties/${PROPERTY_ID}`,
      metrics: [{ name: "activeUsers" }],
      dimensions: [{ name: "unifiedScreenName" }],
      limit: 10,
    });
    const rows = (response.rows || []).map((row) => ({
      page: row.dimensionValues[0].value,
      activeUsers: row.metricValues[0].value,
    }));
    return { content: [{ type: "text", text: JSON.stringify(rows, null, 2) }] };
  }
);

const transport = new StdioServerTransport();
await server.connect(transport);

Lisää GA4-serveri MCP-konfiguraatioon

Päivitä ~/.claude/.mcp.json:

{
  "mcpServers": {
    "google-search-console": {
      "command": "google-searchconsole-mcp",
      "args": [],
      "env": {}
    },
    "google-analytics": {
      "command": "node",
      "args": ["C:/Users/KÄYTTÄJÄNIMI/.claude/mcp-servers/ga4/index.js"],
      "env": {
        "GA4_PROPERTY_ID": "VAIHDA_OMA_PROPERTY_ID"
      }
    }
  }
}
Vaihda KÄYTTÄJÄNIMI ja GA4_PROPERTY_ID omiisi.

OSA 3

Mitä tällä voi tehdä?

Kun molemmat yhteydet toimivat, voit pyytää AI:ta tekemään analyysejä jotka normaalisti vaatisivat manuaalista työtä:

"Top 10 sivua orgaanisesta hausta"
Hakee GSC-datan ja listaa parhaat sivut
"Vertaa tammikuun ja helmikuun liikennettä"
Ajaa kaksi GA4-raporttia ja vertailee tuloksia
"Avainsanat joilla CTR on alle 3%?"
GSC-analyysi, löytää optimointimahdollisuudet
"Onko sivustolla reaaliaikaista liikennettä?"
GA4 realtime -raportti sekunneissa
"Yhteenveto viimeisen kuukauden SEO-datasta"
Yhdistää GSC + GA4 datan samassa vastauksessa
AI osaa yhdistää molempien lähteiden dataa samassa vastauksessa. Sinun ei tarvitse vaihtaa välilehtien välillä.

Vianmääritys

GSC ei toimi

  • Poista OAuth-token ja kirjaudu uudelleen
  • Tarkista asennus: npm list -g google-searchconsole-mcp

GA4 ei toimi

  • Yleisin syy: gcloud-tunnukset vanhentuneet — aja gcloud auth application-default login
  • Tarkista että Analytics Data API on käytössä Google Cloud Consolessa
  • Property ID pitää olla pelkkä numero, ei UA- alkuinen

MCP-serveri ei lataudu

  • Käynnistä Claude Code uudelleen (/exit ja claude)
  • Tarkista .mcp.json -syntaksi JSON-validaattorilla
  • Testaa manuaalisesti: node ~/.claude/mcp-servers/ga4/index.js
